Here are the rules for labeling:
- If the PR fixes a bug, label it with "bug".
- If the PR updates dependencies, label it with "dependencies".
- If the PR is about documentation, label it with "documentation".
- If the PR adds a new feature, label it with "enhancement".
- If the PR updates GitHub Actions, label it with "github_actions".
- If the PR updates python code, label it with "python".
- If you can't find an appropriate labels for the PR, follow the previous instruction that starts with "IMPORTANT:".
- If the PR adds new test files, label it with `category:new-test`.
- If the PR modifies test infrastructure (e.g., test runner, fixtures, CI), label it with `area:test-infra`.

Here are the guidelines for checking if the PR is following the guidelines:
- The "statusCheckRollup" in the pull request details may help you to identify if the PR is following some of the guidelines (e.g. CLA compliance).
- If new test files are added, check that they contain comments or documentation explaining which part of the UCP specification they are validating. If not, ask the author to add this information.
- Check if the PR description mentions the UCP specification version that the tests are being added or updated for. If not, ask the author to specify it.
